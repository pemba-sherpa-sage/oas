Accounts Payable tax schedule maps

Tax schedule maps are used to associate tax schedules with one or more combinations of contact tax groups, account label tax groups, and item tax groups. These maps drive the taxes that are applied to a line item.

AR tax schedule maps

An Accounts Receivable tax schedule map associates a tax schedule with customers (through a contact tax group) and account labels (through an account label tax group). These maps drive the taxes that are applied to a line item. The tax schedule contains the tax detail records, which define the amount of tax to apply.

List AR tax schedule maps

Returns a collection with a key, ID, and link for each Accounts Receivable tax schedule map. This operation is mostly for use in testing; use query to find objects that meet certain criteria and to specify properties that are returned.

Accounts Receivable tax schedules

A tax schedule identifies the tax details that are applied when the schedule is referenced by a tax schedule map. For example, a tax schedule named "Purchase Goods Standard Rate" could include one tax detail named "Purchase Goods Standard Rate". When this schedule is referenced, the tax detail is applied. See Tax schedules—Taxes application for more information.

Create an Order Entry tax detail

Creates a new Order Entry tax detail.

Request Body schema: application/json
id
required
string

Unique identifier of the Order Entry tax detail.

status
string (status)
Default: "active"
Enum: "active" "inactive"

Object status.

taxUniqueId
string

A valid identifier for Australia GST, South Africa VAT, or United Kingdom VAT compliance. Valid identifiers are predetermined by internal Sage requirements.

taxRate
string
Enum: "standard" "reduced" "exempt" "zero" "federal" "provincial"

Tax rate.

Valid values

  • standard - Standard tax rate.
  • reduced - Reduced tax rate.
  • exempt - Exempt from tax rate.
  • zero - Zero tax rate.
  • federal - Federal tax rate.
  • provincial - Provincial tax rate.
amountToTax
string
Enum: "fullAmount" "amountWithinRange"

Amount to include in taxable amount.

** Valid values **

  • fullAmount - Include the cost from zero up to the maximum taxable amount.
  • amountWithinRange - Include the amount between the minimum and maximum taxable amount.
description
string

The description of the Order Entry tax detail, which appears in the transaction subtotals if configured.

taxPercent
number

Tax rate percent.

object
reverseCharge
boolean
Default: false

Reverse charge flag.

useExpenseAccount
boolean
Default: false

Set to true to assign taxes to the expense account.

object (Account label)

Account label.

object
object

GL account to which this tax is to be assigned.

object

Different tax solutions are available for different countries. Examples include Australia GST, South Africa VAT, and UK VAT.
